## 一、AI 帮了什么

### 1.1 简历重写（Hermes Agent）

- 从旧版 PDF 提取文本，按 Web3 Ops 方向重构结构：删除嵌入式项目细节 / 重构德业经历描述 / 新增 Monad 实训经历
- 将「核心能力」表格转为段落式「自我评价」，风格更贴合 Web3 团队阅读习惯
- 全程对话式迭代：改完一段确认一段，不用一次性给完整需求

### 1.2 项目调研与内容产出（anysearch + Hermes）

- Casplus 白皮书分析：用 anysearch 提取 Tokenomics / NFT / Data Collection 等关键页面，AI 汇总要点，减少 2-3 小时手动阅读
- 链上数据查询：通过 RPC 调用直接拉取 Monad Testnet 合约的部署状态和交易回执，不需要手动翻浏览器
- 长文初稿生成：从调研数据直接产出文章骨架，人工在此基础上加判断和语气

### 1.3 Git 仓库管理（终端命令 + 人工确认）

- commit 信息拟定、remote 设置、branch 重命名（master → main）、force push 等操作由 AI 执行命令

### 1.4 简历平台调研（anysearch）

- 搜索 Magic Resume（8.8k ⭐）、web3cv.io 等简历工具的信息、功能和许可协议

### 1.5 封面图尝试（硅基 FLUX Qwen-Image）

- 按 VI 色板写 2 版封面图提示词，调用 API 出图，人工验收后判定 Qwen-Image 风格不达标，转到 Grok

## 二、人类删改 / 核查了什么

### 2.1 简历内容（5 处人工判断）

| 原文（AI 建议） | 人工修改 | 理由 |
|---|---|---|
| 保留嵌入式项目（BMS/无人机/智能车） | **全部删除** | Ops 方向不需要展示 STM32 或 RTOS 能力，反而会分散 HR 对社群/内容的注意力 |
| 德业经历用技术术语描述（逆变器型号/BMS协议） | **改为社群规模和多语种沟通为主** | Web3 Ops 更看重管理规模和跨文化能力，而非具体技术细节 |
| 语言能力写「基础德语/西班牙语」 | **改为「售后场景实战沟通」** | 需要准确描述水平——不是流利但能干活 |
| 核心能力列成表格 | **改为段落式自我评价** | 字数更少但信息量更密，读起来像个人而非技能清单 |
| 照片保留 | **删除** | 非必要信息，且公开渠道不建议放照片 |

### 2.2 Casplus 项目分析（3 处人工判断）

- **「0 团队 0 投资人」是好是坏**：AI 倾向于肯定，但人工判断指出创始团队收入来源是黑箱，匿名团队在机器人赛道存疑
- **NFT 流动性风险**：AI 只列出了地板价 0.25 ETH，人工补充了真实出价 0.0088 ETH（差 28 倍）的对比
- **风格控制**：AI 初稿有明显 AI 味（「底层的经济逻辑是」→ 人工改为「说白了就是」），每段收束太完美需要打碎

### 2.3 封面图（2 轮人工判断）

- Qwen-Image 第一版偏日系可爱 → 人工确认风格不对，调整 prompt 重出
- 第二版接近可用但未达到地下漫画涂鸦感 → 人工判断 Qwen 模型上限，决定后续用 Grok

### 2.4 Git 操作（每次都是人工判断后授权）

- `git push --force`、目录删除等破坏性操作必须经过用户确认
- 提交信息由 AI 草拟但用户决定最终版本

## 三、哪些不能交给 AI

### 3.1 审美判断（不可替代）

- **画面风格是否「对」**：AI 可以列出风格选项，但「这张偏日系可爱了、我想要粗线条涂鸦」是主观审美判断，AI 做不到
- **文字是否「像人写的」**：AI 能检测 AI 味（dbs-ai-check），但「改成更像 Pan 的口吻」需要了解这个人的写作习惯和性格

### 3.2 价值判断（不可替代）

- **项目该不该跟**：AI 可以列出 Casplus 的优点和风险，但「值不值得花时间写」是取决于个人目标和判断的
- **什么经历对 Web3 简历有用**：AI 倾向于保留所有经历，但人需要判断「嵌入式项目会分散 Ops 方向的注意力」

### 3.3 身份展示策略（不可替代）

- **用什么名字投简历**：本名还是花名？取决于投递的公司类型和隐私偏好——AI 不知道这个人的行业关系和风险承受能力
- **哪些经历加权、哪些淡化**：取决于你对目标岗位的判断，不是 AI 能决策的

### 3.4 破坏性操作的最终确认（不可替代）

- `git push --force` 重写远程历史、删除目录——AI 可以执行但最终 YES/NO 必须由人按

## 四、协作效率总结

| 环节 | AI 贡献 | 人工贡献 | 效率提升 |
|---|---|---|---|
| 简历重写 | 初稿生成 / 结构调整 | 内容删减 / 语气校准 | ~4x |
| 项目调研 | 白皮书摘要 / 数据查询 | 结论判断 / 风险分析 | ~5x |
| 长文写作 | 骨架搭建 / 初稿 | 风格控制 / 事实核实 | ~5x |
| 封面生图 | 按 prompt 出图 | 风格确认 / 模型决策 | ~2x（需迭代） |
| Git 管理 | 命令执行 / 提交拟定 | 操作确认 / 分支策略 | ~3x |

**结论：AI 能完成约 70% 的信息处理和初稿生成，但剩下的 30%（审美 / 判断 / 决策 / 确认）必须由人完成。** 这不是 AI 的能力问题，而是这些环节天然依赖人的主观标准和风险承担。Week 1 和 Week 2 的实践证明了「AI 初稿 + 人工把关」是最有效的协作模式。
