## 执行流程

### 活动前 60 分钟

- 主持人 + 嘉宾进入 Space 测试麦克风和网络
- 场务在评论区发布活动链接 + 提醒倒计时
- 主持人确认串场词和核心问题卡片

### 活动当天流程

| 时间 | 环节 | 执行要点 |
|---|---|---|
| 0:00–0:05 | 开场 | 主持人自我介绍 + 简单背景（30 秒），然后点明主题 |
| 0:05–0:15 | 主题分享 | 分享人讲 8–10 分钟，留 2–3 分钟给主持人追问 |
| 0:15–0:25 | 圆桌对话 | 主持人读问题，嘉宾 + 学员各答 1-2 分钟 |
| 0:25–0:35 | 互动环节 | 主持人念场景，场务在评论收集答案并挑 3 条读 |
| 0:35–0:50 | 自由问答 | 场务从评论区筛选 3-5 个问题，主持人点名提问 |
| 0:50–1:00 | 收尾 | 主持人总结 3 个 takeaway + 读 CTA |

### 主持人串场关键句

- 开场：今天不聊抽象的技术，就聊一件事——Agent 怎么在我们签名的之前确保操作是对的
- 转分享：那这个问题现在有人在做吗？有的。今天请到的嘉宾是 Moss 项目的 contributor
- 转互动：接下来不只有嘉宾讲，大家也可以参与——我给大家一个场景
- 收尾：总结今天的三件事：理解问题、看到一个解法、知道怎么参与

## 风险预案

| 风险 | 可能性 | 应对方案 |
|---|---|---|
| 嘉宾临时无法参加 | 中 | 提前录制 5-8 分钟的主题介绍视频或录音，作为替代素材；改为主持人 + 学员对谈模式 |
| Space 没人说话 | 低 | 场务提前准备 3 个备用问题，主持人在沉默时主动点名互动环节答对的观众分享 |
| 网络中断/卡顿 | 中 | 主持人准备 2-3 个「过渡段」——读评论区的优质见解、播报活动规则、读一条有趣的相关推文 |
| 观众太少 | 低 | 预设最低规模 5 人即可执行；如果低于 3 人直接转为录播，整理成 Thread 发布 |
| 出现恶意发言或广告 | 低 | 场务标记并警告，主持人不直接回应。X Space 支持移除发言者 |
| 核心讨论跑题 | 中 | 主持人用过渡句拉回正题：「这个方向很有意思，我们后面可以单独聊，先回到刚才那个问题」 |
