# Space 内容设计（Run of Show）

## 活动标题

**AI Agent 怎么在链上安全地花钱？——当自动化遇到真金白银**

## 活动简介

Agent 正在变得越来越自主——能帮你查行情、跟策略、甚至执行交易。但链上交互有一个核心矛盾还没解决：Agent 可以组交易但不能签名，你可以签名但不一定每次都能看懂 calldata 里的一行行数据。万一 Agent 填错了一个小数位或者地址，签名就是真金白银的损失

这次 Space 会从 Moss 这个开源项目出发，讨论 AI Agent 在链上交互时的安全边界在哪，Moss 怎么用「模拟验证」来解决这个矛盾，以及作为普通用户或内容创作者怎么看待和参与这个方向

## 核心讨论问题

**问题 1：Agent 替我们执行链上操作，到底是更方便了还是更危险了？**
引导方向：对比「人工检查每一笔交易」和「Agent 自动执行」的安全边界。是否需要一个中间层来平衡效率和风险？Moss 的 Plan + Simulate 机制是否是这个中间层的可行答案？

**问题 2：Moss 选择「不签名不发送」作为设计原则，是不是意味着 Agent 的自主性被削弱了？**
引导方向：Agent 的自主性和安全如何取舍。Moss 把最终决定权留给用户，是保守还是必要？在未来，这个边界会不会移动？

**问题 3：对于 Ops / Research 方向的人，不写代码怎么参与这类安全基础设施项目？**
引导方向：文档、教程、测试案例、社区教育——非代码贡献在早期开源项目里的价值

## 互动设计

**「如果你是 Agent，你要怎么证明你靠谱？」填空式造句**

主持人给一个场景：Agent 被要求将 100 MON 换成 USDC，但当前链上价格波动剧烈。

让观众在评论区填空：

> 「我会先＿＿＿＿，然后＿＿＿＿，最后才请你签名＿＿＿＿」

预设 3 个正确答案：
- 先检查链上实时价格
- 然后模拟这笔交易看看实际滑点
- 最后告诉你预期结果并等你确认

最快答对 3 条的观众，可以获得主持人/嘉宾的 Moss 项目 Contributor 入门指导一次
