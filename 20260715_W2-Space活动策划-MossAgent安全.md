# Monad Space 活动策划案：「AI Agent 怎么在链上安全地花钱？」

## 活动主题

**AI Agent 怎么在链上安全地花钱？——从 Moss 开源项目聊 Agent 与 Web3 的交叉点**

## 活动目标

1. 让参与者理解 AI Agent 在链上交互时的核心安全矛盾（Agent能跑但人签名、人能签名但不一定看懂）
2. 通过 Moss 这个真实开源项目，展示一种可行的解决方案（Plan + Simulation 双重验证）
3. 吸引对 AI Agent + Web3 交叉领域感兴趣的 Builder，为 Moss 项目引流潜在贡献者

## 目标听众

- Monad 训练营的学员（为主，约 30-50 人）
- 对 AI Agent 开发感兴趣的 Web3 Builder
- 想了解怎么参与开源但还没开始的初学者

## 核心问题

AI Agent 正在变得越来越自主，但链上交互的安全机制还没跟上。让 Agent 直接调合约发交易，一旦 calldata 组错了、小数位算错了、地址填错了，钱就是真金白银地丢。这个矛盾怎么解？

具体来说：

- 能不能让 Agent 在签名前自动模拟一遍交易，确认效果再签？
- 开源社区在这个方向做了什么？Moss 的方案是什么？
- 作为 Ops/Research 方向的人，不写代码能怎么参与这类项目？

## 举办理由

1. **Monad 训练营的学员天然关心链上交互效率**——Week 1 每个人都部署过合约，知道 gas 和交易验证的痛点
2. **AI Agent 赛道在 Monad 生态的关注度在上升**，但中文内容几乎没有。Moss 是 Monad 生态的原生项目，选它作为案例天然契合
3. **OPS 方向的学员需要展示自己能策划和组织社区活动**，这是一个低成本的实践机会（线上 Space，不需要场地和预算）
4. Moss 目前 72 Star，社区小但活跃，maintainer 响应快，如果能邀请到项目作者 nishuzumi 来 Space 分享，对双方都是曝光

## 活动形式

X Space，60 分钟

| 环节 | 时长 | 内容 |
|---|---|---|
| 开场 | 5 min | 主持人介绍主题和嘉宾 |
| 主题分享 | 20 min | Moss 项目简介 + Agent 安全的思路 |
| 圆桌/对谈 | 15 min | 嘉宾 + 2 位训练营学员聊 Agent × Web3 |
| 自由问答 | 15 min | 开放麦，参与者提问 |
| 结尾 | 5 min | 总结 + 招募贡献者 |

## 参考案例

1. **「Moss Architecture Walkthrough」**（Moss 项目官方在 Discord 举办的线上技术分享会，围绕 ADR 和 Capability 架构展开讨论）。参考了它的环节结构和时长控制，但本方案更侧重新手友好度，不会深入 ADR 细节

2. **「Agent 与链」系列 Space（由 univalid 组织的 X Space 系列）**——每期邀请一位 AI Agent + Web3 交叉方向的项目方或开发者做分享，模式是 20 分钟分享 + 20 分钟 Q&A，本方案参考了它的节奏设计

## 备注

- 建议在每周末晚 8 点（北京时间）举办，与训练营作业节奏对齐
