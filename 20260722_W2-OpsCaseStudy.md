# Week 2 Ops Case Study — AI Agent 链上安全 Space 策划

## 一、活动简介

**主题：** AI Agent 怎么在链上安全地花钱？——当自动化遇到真金白银

**定位：** 一场面向 Monad 训练营学员和 AI Agent × Web3 方向 Builder 的线上 Space，目标不是讲技术原理，而是用 Moss 开源项目作为案例，让参与者理解 Agent 链上交互的核心安全矛盾，以及不写代码的人怎么参与这类项目

**形式：** X Space，60 分钟

## 三、核心运营思考

**本周最重要的一个运营判断：为同一个活动生产不同类型的内容，而不是分散做多个不相关的事情。**

从策划案 → 内容设计（Run of Show）→ 执行预案 → 运营物料（宣传文案/提醒/CTA），这些不是 4 个独立任务，而是**同一个活动在不同阶段的不同产物**。每份产出都建立在上一份的基础上：

- 如果活动主题没想清楚，后面的 Run of Show 就没有方向
- 如果 Run of Show 没设计好互动环节，物料里的 CTA 就编不出来

这条逻辑同样适用于周报和简历——持续产出同一个方向的内容，比在不同方向各做一点，更容易形成作品集。

## 四、AI 协作记录

| 环节 | AI 协助 | 人工判断 |
|---|---|---|
| 活动策划案 | 生成活动框架和参考案例 | 选题决策——为什么选 Moss 而非其他项目；目标听众的范围划定 |
| Run of Show | 3 个讨论问题的初稿、填空互动设计 | 问题顺序的节奏感（从认知矛盾到具体解法）；互动环节的时间和难度设置 |
| 执行预案 | 风险清单的初稿、时间排期表 | 哪些风险是真实的（嘉宾档期/网络卡顿）哪些是多余的；预期目标是否合理 |
| 运营物料 | 文案初稿、提醒话术 | 版本 A 和版本 B 的选择策略——X 用短版、社群用长版；CTA 的语气控制 |
| 简历重构 | 初稿生成、结构建议 | 哪些内容删（嵌入式项目）、哪些内容增（德业社群管理）；自我评价的风格和重点 |

**AI 可以做到的：** 生成结构性初稿（框架/排期/清单）、提供参考方向、优化措辞。

**必须人工判断的：** 选题决策、目标受众划定、内容和自己 IP 风格的对齐、风险的优先级排序。

## 五、Week 3 Role Statement

进入 Week 3 后，我希望继续在 Ops 方向深耕，具体定位为：

**内容创作者 + 社区运营辅助。**

- **持续产出：** 每周至少 1 篇 Monad 生态项目深度拆解 + 对应 X Thread。内容方向从项目分析扩展到社区活动复盘和学习方法论。
- **社区互动：** 在能力范围内参与评论区互动和社区反馈收集，但不强求自己做社群日常运营——这部分需要另一个 Ops 角色来补。
- **开源贡献：** 继续保持对 Moss 的关注，在文档/教程方向做长期贡献，争取在 Week 3 结束时完成一个 merged PR。
- **技能补充：** Week 3 计划学习基础的数据分析能力（X Analytics / 内容数据追踪），让 Ops 产出有数据支撑，而不是只凭感觉。
