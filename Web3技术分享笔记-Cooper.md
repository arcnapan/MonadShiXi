# Web3 Tech Share 笔记 — Cooper

## 第一部分：跟 AI 一起开发

### AI 开发的四个核心能力

**会描述** — 学会写好的 Prompt
- 把你的想法写成明确的目标
- 告诉 AI 用户是谁、场景是什么、约束和验收标准
- 目标：最大化发挥 AI 能力的 Prompt

**会拆解** — 把复杂任务拆成小步骤
- 分成可执行、可理解、可验证的小步骤
- 搞清楚每一步的依赖、风险和产出

**会验证** — 自己参与验证
- 手动测 / 自动化测 / 测试用例
- 留下日志和记录
- **核心：自己把关验收结果，开发出来的东西自己要能判断对不对**

**会设计** — 跟 AI 一起学会设计
- 架构边界、数据流、协作规则、质量体系
- 不只是拿结果，要在过程中学

### AI 不是替你思考，是加速你的能力

- 初級程序员会越来越少，高级工程师 + AI 的组合效率远超多个初级程序员
- 但你**不能脑袋放空**——像发指令一样丢给 AI 拿结果完事，要在过程中学
- 经典段子：有人分享自己的项目，链接是 http://127.0.0.1 — 被嘲笑是 AI 写的。但 Cooper 认为：即使没 AI，你不懂计算机网络也会犯这个错。有 AI 反而让你更快知道问题在哪并纠正

## 第二部分：Web3 开发技术栈

### 区块链是什么

一句话：**异步的、强一致性的、可编程的金融平台**

- 异步：发交易后不能马上拿到结果，要等打包+共识完成
- 强一致性：全网几万个节点，对同一笔交易执行结果完全一致
- 可编程：智能合约能执行你自己设定的逻辑
- 金融平台：Web3 跟金融贴得很近，不管什么玩法多少都跟金融有关系

### Web3 开发的 5 个方向

| 方向 | 做什么 | 适合谁 |
|---|---|---|
| **DApp 开发** | 前端页面 + 钱包连接 + 链上数据展示 | ✅ **最适合新手**，直观感受强，成就感强 |
| **智能合约开发** | Solidity / Vyper，链上状态管理 + 协议规则 | 用 Hardhat/Foundry 框架，OpenZeppelin 模板 |
| **钱包开发** | 密钥管理、签名、账户抽象、防钓鱼/防重放 | 责任重——间接管理所有资金 |
| **节点开发** | P2P网络、共识规则、存储、RPC | Go/Rust，偏底层，比较"苦"但好玩 |
| **安全审计** | 代码 review + 审计报告 | 垂类窄地位高，保护项目方资金安全 |

### DApp 开发技术栈

- **前端**：React / Vue
- **链交互**：ethers.js / web3.js
- **API**：以太坊 JSON-RPC
- **数据**：事件机制 → 不够用就上 Indexer
- **测试**：单元测试 + E2E
- **核心**：你看到的是一个 Node URL，背后的区块链网络对 DApp 开发者是透明的

### 用户视角 vs 开发者视角

```
用户 → DApp（前端/App）→ RPC接口 → 执行层（EVM）→ 共识层 → P2P网络
```

对 DApp 开发者：只需要关心 Node URL 和以太坊的协议规范就行

### 一笔交易的完整生命周期

DApp 编码调用 → 估算 Gas → 构造交易 → 钱包签名 → RPC 广播 → 加入交易池 → P2P 广播 → 打包区块 → 执行交易 → 更新状态 → 共识达成最终性 → DApp 读取结果/事件
